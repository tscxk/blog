JVM面试题

综合

1. 说说JVM的主要组成部分和流程

JVM主要包含两个子系统和两个组件，两个子系统分别是Class loader(类装载)、Execution engine(执行引擎)；两个组件分别是Runtime data area(运行时数据区)、Native Interface(本地接口)。

  • Class loader(类装载)：根据类的全限定名来装载class到运行时数据区域的方法区中
  • Runtime data area(运行时数据区)：即我们常说的JVM的内存
  • Execution engine(执行引擎)：负责将字节码文件“翻译”成对应平台上的机器指令并执行
  • Native Interface(本地接口)：和其它编程语言交互的接口

流程：编译器首先将Java代码转换成字节码。类加载器负责将字节码加载到运行时数据区域的方法区中。由于字节码是跨平台语言，因此需要解释器引擎根据平台翻译成具体的机器指令再交由cpu执行。本地库接口则是方面java代码进行系统调用或者其他函数的调用。

方法区相关

1. 永久代和元空间的联系和区别

方法区是虚拟的规范，是逻辑上的定义

永久代是JDK8之前的方法区实现，元空间是JDK8以及之后之后的方法区实现

核心区别如下（主要是位置区别）：

  • 永久代：位于Java堆内存中，其大小收到堆容量的限制，其垃圾回收受到Java堆垃圾回收机制管理
  • 元空间：从Java堆中移除转而使用本地内存，Java堆垃圾回收机制不再负责其垃圾回收，但是其垃圾回收仍然由JVM负责

其他改进如下：

jdk7：将字符串常量池（interned strings）和类静态变量（class statics）转移到Java堆中

jdk8：元空间彻底取代永久代

2. 为什么要替换永久代？

  • 消除永久代的大小限制：永久代经常因为加载的类过多（尤其是在应用服务器、大量使用反射、动态代理的场景）而出现OutOfMemoryError，使用本地内存可以大幅减少这类问题。（永久代的调优通常是十分困难的）
  • 垃圾回收优化：永久代的垃圾回收通常和老年代的垃圾回收（通常是Full GC）捆绑在一起，而Full GC的代价非常高。元空间在本地内存可以使用高效、独立的垃圾回收
  • 未来改进优化：将HotSpot与JRockit虚拟机代码库合并（两者都是Oracle的），JRockit本来就没有永久代的概念
